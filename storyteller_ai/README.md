======================

Generated by bootstrap_backend.py

Quickstart
----------

1. Run `install_dependencies.bat` once.
2. The installer creates a desktop shortcut for `launch_storyteller_ai.bat` automatically.
3. Double-click the desktop shortcut or `launch_storyteller_ai.bat` to start the backend.
4. For a one-click offline startup with Ollama and Llama2, run `launch_storyteller_ai_ollama.bat`.
5. Open your browser at:

    <http://127.0.0.1:8000/>

6. Use the frontend UI or the `/sessions` and `/gm` endpoints.

Local LLM Support (Offline Operation)
-------------------------------------

The backend now defaults to Ollama at runtime. If Ollama is running locally, the app will use it unless you explicitly change the provider in Setup or via environment variables.

Quick Setup for Ollama (Recommended for Offline Use)
---------------------------------------------------

Run this ONE command to set everything up automatically:

```powershell
.\setup_ollama.ps1 -Action install -Model llama2:7b
```

Then create `.env` file with:

```env
LLM_PROVIDER=ollama
OLLAMA_URL=http://127.0.0.1:11434
OLLAMA_MODEL=llama2:7b
```

See [OLLAMA_QUICKSTART.md](./OLLAMA_QUICKSTART.md) for detailed instructions.

Available Providers
-------------------

Set `LLM_PROVIDER` to one of:

- `ollama` — uses a local Ollama HTTP endpoint (recommended for offline use)
- `openai` — uses `OPENAI_API_KEY`
- `anthropic` — uses `ANTHROPIC_API_KEY`
- `mock` — uses a packaged offline mock provider

Example for Ollama:

    set LLM_PROVIDER=ollama
    set OLLAMA_URL=http://127.0.0.1:11434
    set OLLAMA_MODEL=llama2:7b
    uvicorn backend.main:app --reload

Example for mock mode:

    set LLM_PROVIDER=mock
    uvicorn backend.main:app --reload

Packaging as a Single Executable
---------------------------------

The current codebase is now scaffolded for packaging with PyInstaller.

Recommended next step:

1. Run `build_storyteller_ai_exe.bat` from the repository root.
2. The script installs PyInstaller into the project venv if needed.
3. The build bundles the FastAPI app and the `frontend/` static files.
4. Document data is written next to the exe in a local `data/` folder at runtime.

Typical packaging flow:

    build_storyteller_ai_exe.bat

Output location:

    dist\StorytellerAI.exe

Frontend
--------

Start the backend and open the UI in a browser at:

    http://127.0.0.1:8000/

The frontend includes:

- session creation
- chat / GM step input
- PDF upload for document ingestion
- document retrieval query

Document ingestion API
----------------------

PDF extraction now includes OCR fallback for scanned/image-only pages.

OCR setup (for scanned PDFs):

1. Install Tesseract OCR on Windows.
2. Ensure `tesseract.exe` is on `PATH`, or set `TESSERACT_CMD` to the full executable path.
3. Install Python dependencies in `requirements.txt` (`pytesseract`, `Pillow`).

POST /documents/upload

- form field: `file` (PDF file)

GET /documents/list

- returns loaded documents and sizes

POST /documents/retrieve

- request body: `{ "query": "..." }`
- returns the best matching text from uploaded documents

Endpoints
---------

POST /sessions/create

- Request body: `{ "mode": "group" }`
- Response: `{ "session_id": "...", "mode": "group" }`

POST /gm/step

- Request body: `{ "session_id": "...", "message": "Your action here" }`
- Response: `{ "text": "...", "mode": "group" }`
